Overview

Autodesk BIM 360 Services for Construction is among the fastest growing products in Autodesk’s history. BIM 360 APIs and SDK allows developers to extend the capabilities by harnessing the power of web services to transforms the way Construction project data is collected, connected, visualized and managed in the cloud. As a cloud platform, Glue and Field are built for integration with third party systems through our Web Services API. Integration can be as simple as embedding the BIM 360 3D viewer into an application (think of embedding a Youtube video in a web page) or as sophisticated as linking BIM object data to project workflows. Integrations are built on the premise of semantically linking, rather than moving data.

Features

  • BIM 360 is an open platform with APIs that provides a mechanism for integrating disparate systems in a secure and scalable manner.
  • Connect BIM and critical business workflows with bidirectional integration for Autodesk® and third-party project management and ERP solutions.
  • REST and JavaScript API makes it easier to create applications.
  • Our Viewer Component provides access to model and model components data in your web app - no need for desktop CAD software installations.
  • 2D and 3D models come to life right within your browser - view detailed textures, vivid visualizations, smooth navigation, and in-depth design data.
  • Browser compatibility with Internet Explorer, and Firefox.
  • Visualize and interact with more than 60 design file formats in your web app; no plug-ins or downloads required.

Play with the APIs

Pretty exciting, right? Eager to learn some APIs and get it running on your own website? Here are some live samples you can play with to understand the workflow and the client APIs.

The API comes in two parts -

  • A (server-side) REST API that allows you to translate any of over 70 file formats into a common (JSON) file format. The data extracted from the original file is the geometry of every element in the model and the metadata that was attached to each of those model elements.
  • A (client-side) JavaScript API that simplifies displaying and interacting with a model that has been processed by the translator. The UI for the viewer defined by the API can be heavily customized, viewer actions can be automated (zoom, pan rotate, explode, etc), and callbacks are available for user-driven events (e.g. a mouse click on a specific model element, a change in camera position, etc).

Play with the following live applications first to understand the server side REST API, practice some Client side JavaScript APIs, and then setup your first 3D model viewer web page with the utility live sample. Make sure you check the source code of these live samples.

Step-By-Step Guide

Step 1: Sign in and create an application

To use the Autodesk View and Data API, first create an Autodesk Account. To do so, click on "Get Started" from navigation bar and fill out the information.

The developer portal will send you an email with login information. Use the URL in the email to log in to developer.autodesk.com. Once you're signed in, you can create an application.

  • Click on "Create an App" button on navigation bar or in "My Apps" page.
  • Enter your application name, description and redirect URL(callback URL).
  • Select "View and Data API" API.

Note: the Redirect URL is for OAuth2 2 leg authentication. Don't worry if you do not have a callback set up yet, but a value is required in this field to proceed.

Once you have an application set up, you will see a Consumer Key and Consumer Secret in your newly created app page or in registered email. Note what these are, you will need them for the next step.
